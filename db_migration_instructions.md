### Пошаговая инструкция по переносу `database.db`

Чтобы дать вам максимально точные и безопасные команды, мне нужна следующая информация. Пожалуйста, замените значения в `<...>` на ваши реальные данные:

1.  **IP-адрес старого сервера** (например, `192.168.1.100`).
2.  **Имя пользователя (логин)** для подключения к **старому** серверу (например, `root` или `ubuntu`).
3.  **Полный путь к файлу `database.db`** на **старом** сервере (например, `/home/user/project/database.db`).
4.  **Полный путь к директории проекта** на **новом** сервере, куда нужно положить файл (судя по вашим файлам, это, вероятно, `/root/ati_antares_kWork`).

Выполнять все команды нужно на **новом** сервере через PuTTY.

**Шаг 1: Остановите приложение на новом сервере**

Это необходимо, чтобы приложение не использовало и не блокировало файл базы данных во время его замены.

```bash
# Замените 'ati-bot' на имя вашего процесса в PM2, если оно другое
pm2 stop ati-bot
```

**Шаг 2: (Опционально, но рекомендуется) Сделайте резервную копию текущей БД на новом сервере**

Если на новом сервере уже есть файл `database.db` (даже если он пустой), лучше его сохранить на всякий случай.

```bash
# Перейдите в папку проекта
cd /root/ati_antares_kWork

# Переименуйте текущий файл
mv database.db database.db.backup
```

**Шаг 3: Скопируйте базу данных со старого сервера**

Мы будем использовать команду `scp` (secure copy), которая работает через SSH. Выполните эту команду на **новом** сервере. Она "заберёт" файл со старого.

Замените значения в `<...>` на ваши:

```bash
scp <логин_на_старом_сервере>@<IP_старого_сервера>:<полный_путь_к_database.db_на_старом> /root/ati_antares_kWork/
```

**Пример команды:**

```bash
scp root@37.143.13.205:/root/ati_antares_kWork/database.db /root/ati_antares_kWork/
```

*   После выполнения этой команды система попросит вас ввести **пароль от старого сервера**.
*   Файл `database.db` будет скопирован в директорию `/root/ati_antares_kWork/` на **новом** сервере.

**Шаг 4: Проверьте, что файл успешно скопирован**

Убедитесь, что файл `database.db` появился на новом сервере и его размер не нулевой.

```bash
# Проверяем наличие и размер файла
ls -lh /root/ati_antares_kWork/database.db
```

Вы должны увидеть информацию о файле, включая его размер, который должен совпадать с размером на старом сервере.

**Шаг 5: Запустите приложение на новом сервере**

Теперь, когда база данных на месте, можно запускать ваше приложение.

```bash
pm2 restart ati-bot
```

**Шаг 6: Проверьте логи приложения**

Сразу после перезапуска проверьте логи, чтобы убедиться, что приложение запустилось корректно и не выдает ошибок, связанных с базой данных.

```bash
pm2 logs ati-bot --lines 50
```

Если в логах нет ошибок, значит, перенос прошел успешно, и приложение работает с данными со старого сервера.
